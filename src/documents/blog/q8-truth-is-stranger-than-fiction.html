This week I was looking at a common mistake with regards to JavaScript falsey values. If you remember your JavaScript theory you'll know that there's a lot of different falsey values in JavaScript.

Something that has been coming up a lot in JavaScript design patterns of recently is the use of the double-not operator, or !!.

The double-not operator is useful as it will force a boolean in a scenario like:

  • Checking a boolean result as a boolean (ie - not as an assumed boolean)
  • Returning a result from a function

So next time you see a !! in your JavaScript know it's there to force a type to be a boolean.
