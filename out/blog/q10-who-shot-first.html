The answer I was looking for to this weeks question is Han because after all Han always shot first ;). Technically the answer to this weeks question is it depends because timers work slightly different in each JavaScript runtime but to be closes to the spec you're looking at Han.

The reason for this is that according to the spec the minimum for a timeout in JavaScript is 4 milliseconds and any timeout that is less than 4 milliseconds will be increased to 4 milliseconds. This means that the two functions have the same timeout value and they will be executed in the order they are added to the timeout queue.

So next time you're using small timeouts be sure to test them across your different runtimes so that they actually work as expected.
